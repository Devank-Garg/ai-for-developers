# Resources: Terminology

These are curated external resources if you want to go deeper on any concept from this module. None are required — the concepts file covers everything you need to continue the course.

---

## Foundational overviews

**Andrej Karpathy — Intro to Large Language Models**  
https://www.youtube.com/watch?v=zjkBMFhNj_g  
A 1-hour YouTube talk by one of the field's best explainers. Covers what LLMs are, how they work, and where they're going. The single best introduction that exists. Watch this if you haven't already.

**Simon Willison — Things I've learned about LLMs**  
https://simonwillison.net/2023/Aug/3/weird-world-of-llms/  
A practitioner's perspective on how LLMs actually behave — including the parts that are strange and counterintuitive. Written by a developer for developers.

---

## Specific terms

**Tokenisation**  
Tiktokenizer (https://tiktokenizer.vercel.app/) — an interactive tool that shows you exactly how OpenAI's tokeniser splits any text you enter. Useful for building intuition about tokens vs words, and for understanding why non-English text tokenises differently.

**Embeddings**  
"Word2Vec explained" — search for any of several good visual explainers on this. The concept of words-as-vectors-in-space, and the famous example that "king - man + woman ≈ queen", is the clearest introduction to what embeddings represent.

**Hallucination**  
"Stochastic Parrots" (Bender et al., 2021) — the paper that introduced much of the current discourse around LLM limitations, including the argument that models are pattern-matching on form rather than understanding meaning. The paper is academic but readable.

## Reference

**ML Glossary (Google)**  
https://developers.google.com/machine-learning/glossary  
A comprehensive reference glossary from Google's ML education team. Useful as a lookup when you encounter unfamiliar terms later in the course.

**The Illustrated Transformer (Jay Alammar)**  
https://jalammar.github.io/illustrated-transformer/  
Not required reading at this stage, but bookmark it. It's the best visual explanation of transformer architecture, which will come up heavily in Phase 3 and Phase 4.

## If you want to go much deeper

**"Attention Is All You Need" (Vaswani et al., 2017)**  
The original transformer paper. Not required reading — the math is dense. But if you want to understand where modern LLMs came from, this is the source.

**fast.ai Practical Deep Learning for Coders**  
https://course.fast.ai/  
A free, code-first course that goes deeper on training, datasets, and model evaluation. If you finish this course and want to understand the mechanics more deeply, fast.ai is a strong next step.
